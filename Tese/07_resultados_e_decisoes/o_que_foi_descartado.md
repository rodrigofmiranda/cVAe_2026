## Síntese

Até aqui, perderam força suficiente para não serem tratadas como eixo principal:

- leitura de que erro médio bastaria para validar o twin
- decoder gaussiano simples como solução dominante do gargalo difícil
- hipótese de que o problema remanescente era global e difuso
- promoção automática das variantes `shape` mais complexas
- uso de `G6` como definição principal de validação
- equiparar ganho operacional geometry-biased a baseline científica limpa

## Descartes práticos relevantes

- aumentar complexidade geométrica sem controle científico claro
- usar apenas capacidade latente como resposta principal
- transportar diretamente o melhor proxy `full_square` para `full_circle`
  como se fosse baseline inequívoca
